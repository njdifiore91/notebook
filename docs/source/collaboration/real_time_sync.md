# Real-Time Synchronization

## Overview

Jupyter Notebook v7 introduces comprehensive real-time collaborative editing capabilities powered by the [Yjs CRDT framework](https://yjs.dev/). This feature enables multiple users to simultaneously edit notebook documents with automatic conflict resolution, providing a seamless collaborative experience similar to Google Docs or other modern collaborative editors.

With real-time synchronization, you can:

- Edit notebook content (code cells, markdown cells) simultaneously with other users
- See other users' cursors and selections in real-time
- View outputs generated by other users' code executions
- Collaborate without edit conflicts or data loss

## How It Works

Jupyter Notebook v7's real-time synchronization is built on the following key technologies:

### Yjs CRDT Framework

The core of the real-time synchronization is powered by [Yjs](https://yjs.dev/), a high-performance implementation of Conflict-free Replicated Data Types (CRDTs). Unlike traditional Operational Transformation (OT) approaches, CRDTs:

- Enable peer-to-peer synchronization without requiring a central authority for conflict resolution
- Guarantee that all clients converge to the same document state regardless of network conditions
- Support offline editing with automatic merging when reconnecting
- Provide efficient binary update protocol for minimal network overhead

### YjsNotebookProvider

The `YjsNotebookProvider` is a custom implementation that bridges between the Jupyter Notebook model and the Yjs document:

- It wraps the notebook model with a Yjs document structure
- Maps notebook cells, metadata, and outputs to appropriate Yjs shared types
- Handles bidirectional synchronization between the notebook model and Yjs document
- Manages WebSocket connections for real-time updates

### WebSocket Communication

Real-time updates are transmitted via WebSocket connections:

- Binary Yjs updates are sent and received through a dedicated WebSocket endpoint
- The y-websocket provider handles connection management and update propagation
- Server-side handlers broadcast updates to all connected clients
- Presence information (cursors, selections) is shared through the awareness protocol

## Configuration

### Enabling Real-Time Synchronization

Real-time synchronization can be enabled or disabled through configuration settings:

```bash
# Enable collaboration in jupyter_server_config.py
c.JupyterNotebookApp.collaborative = True

# Or via command line
jupyter notebook --collaborative=True
```

### WebSocket Provider Configuration

The WebSocket provider can be configured with custom settings:

```python
# In jupyter_server_config.py

# Configure the WebSocket endpoint URL (defaults to auto-detection)
c.JupyterNotebookApp.collaboration_websocket_url = "wss://your-server.example.com/api/yjs"

# Set connection retry parameters
c.JupyterNotebookApp.collaboration_connection_retry_max = 10
c.JupyterNotebookApp.collaboration_connection_retry_delay = 1000  # milliseconds
```

### Persistence Configuration

You can configure how collaborative document state is persisted:

```python
# In jupyter_server_config.py

# Select persistence backend ("file" or "database")
c.JupyterNotebookApp.collaboration_persistence = "file"

# Configure persistence options
c.JupyterNotebookApp.collaboration_persistence_options = {
    "update_interval": 60,  # seconds between persistence operations
    "history_snapshots": 10  # number of historical snapshots to maintain
}
```

## Deployment Scenarios

### Single-Server Deployment

For a basic single-server deployment:

1. Enable collaboration in your configuration
2. Ensure WebSocket connections are properly routed if behind a proxy
3. Configure persistence to match your storage requirements

```bash
jupyter notebook --collaborative=True --collaboration_persistence="file"
```

### JupyterHub Integration

When deploying with JupyterHub:

1. Configure JupyterHub to use Jupyter Notebook v7 with collaboration enabled
2. Ensure authentication tokens are properly passed to the collaboration provider
3. Configure user permissions based on JupyterHub authentication

Example JupyterHub configuration:

```python
# In jupyterhub_config.py
c.Spawner.default_url = '/notebook'
c.Spawner.cmd = ['jupyter-notebook']
c.Spawner.args = ['--collaborative=True']
```

### Custom Deployment

For custom deployments with specific requirements:

1. Configure the WebSocket endpoint URL explicitly if auto-detection doesn't work
2. Set up a database backend for persistence if file-based storage is insufficient
3. Adjust connection parameters based on your network environment

## Troubleshooting

### Connection Issues

If you experience connection problems:

1. Check that WebSocket connections are allowed by any proxies or firewalls
2. Verify the WebSocket URL is correctly configured and accessible
3. Increase connection retry parameters if experiencing intermittent connectivity

```python
# Increase connection resilience
c.JupyterNotebookApp.collaboration_connection_retry_max = 20
c.JupyterNotebookApp.collaboration_connection_retry_delay = 2000  # milliseconds
```

### Synchronization Problems

If changes aren't synchronizing properly:

1. Check browser console for any error messages related to Yjs or WebSocket connections
2. Verify that all clients are connected to the same document
3. Try refreshing the page to establish a new connection
4. Check server logs for any backend errors in the collaboration handlers

### Performance Considerations

For optimal performance with real-time synchronization:

1. Limit the number of simultaneous editors on very large notebooks
2. Consider increasing server resources for deployments with many collaborative sessions
3. Configure persistence intervals based on your editing patterns and reliability requirements

```python
# Optimize for performance with many users
c.JupyterNotebookApp.collaboration_persistence_options = {
    "update_interval": 120,  # less frequent persistence
    "compression": True  # enable update compression
}
```
